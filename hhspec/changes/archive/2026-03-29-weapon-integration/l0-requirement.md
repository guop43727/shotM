# L0 需求文档：武器系统集成功能

## 需求概述

将已完成的武器进化系统集成到游戏主循环中，实现波次选择、掉落集成、视觉反馈等功能。

## 功能需求

### FR-1: 波次选择武器 UI
**优先级**: P0
**描述**: 在每波开始前显示武器选择界面，让玩家从库存中选择武器

**验收标准**:
- GIVEN 玩家完成一波游戏
- WHEN 下一波开始前
- THEN 显示武器选择弹窗，展示库存中的武器
- AND 玩家可以点击选择武器
- AND 选择后自动装备并开始新波次

### FR-2: 武器掉落箱集成
**优先级**: P0
**描述**: 敌人死亡时掉落武器，玩家拾取后自动添加到库存

**验收标准**:
- GIVEN 敌人被击杀
- WHEN 敌人死亡
- THEN 有概率掉落武器箱
- AND 玩家碰触武器箱时自动拾取
- AND 武器添加到 WeaponManager 库存

### FR-3: 合成动画和音效
**优先级**: P1
**描述**: 武器合成时显示动画效果和播放音效

**验收标准**:
- GIVEN 玩家触发武器合成
- WHEN 合成成功
- THEN 显示粒子动画效果
- AND 播放合成音效
- AND 显示合成结果提示

### FR-4: E2E 测试
**优先级**: P1
**描述**: 完善端到端测试覆盖

**验收标准**:
- 测试完整的武器获取→合成→装备流程
- 测试波次选择流程
- 测试掉落拾取流程

## 技术约束

- 纯前端实现，无后端 API
- 使用 HTML5 Canvas 渲染
- localStorage 持久化
- 保持赛博朋克 UI 风格
- 最小化代码量

## 依赖文件

- game.js - 游戏主循环
- weaponManager.js - 武器管理器
- weaponUI.js - 武器 UI
